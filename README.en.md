# Daily Note Automation 📝

> CLI tool that analyzes your git changes and writes them into your Obsidian daily note.

🌐 [한국어로 보기](README.md)

Powered by Claude Code — stop trying to recall "what did I do today?" while writing your daily note.

## ✨ Features

- 🚀 **`dstart`**: create today's daily note + take a git snapshot
- 🏁 **`dend`**: have an AI analyze your git changes and write them into the daily note
- 📊 **`dusage`**: Claude Code token usage and cost stats
- 🔄 **Multi-project**: each project gets its own section and code-changes note
- 📅 **Retroactive mode**: forgot to run `dend` yesterday? It's recovered automatically the next morning.
- 🎯 **Smart baseline**: snapshot is anchored at the *last commit before today midnight*, so commits already made today are still tracked.

```bash
cd ~/work/my-project
dstart
```

## 📋 Requirements

- **macOS** or **Linux** (Windows: WSL recommended)
- **Git** installed
- **Claude Code** installed ([official guide](https://docs.claude.com/en/docs/claude-code))
- **Obsidian** — output uses `[[wiki links]]` and a folder layout (`01_Daily`, `05_CodeChanges`). Other markdown apps can open the files, but wiki links may break.
- (Optional) **jq** — better token parsing (`brew install jq`)

## 🚀 Install

```bash
# 1. Clone
git clone https://github.com/leesj3857/daily-note-automation.git
cd daily-note-automation

# 2. Run the installer
./install.sh
```

The installer asks you:

- **Note language** (Korean / English) — sets the language of daily-note templates and the summary/retrospective notes generated by `dend`. Messages before the language picker are in English; afterwards the installer switches to your chosen language.
- **Folder where notes will live**
  - Use the **vault root**, e.g. `~/Documents/Obsidian/MyVault`
  - Or point to a **sub-folder inside the vault** — if you want to keep work notes under a specific area rather than the vault root, include that sub-folder in the path.
    e.g. `~/Documents/Obsidian/MyVault/Work`
  - The daily folder (`01_Daily`) and code-changes folder (`05_CodeChanges`) are created **directly under** the path you give.
- **Daily-notes folder name** (default: `01_Daily`)
- **Code-changes folder name** (default: `05_CodeChanges`)

> To change the language later, edit `NOTE_LANG` in `~/.daily-noterc` (set to `ko` or `en`), or re-run `./install.sh`.

After install, open a new terminal or:

```bash
source ~/.zshrc          # zsh users
source ~/.bash_profile   # bash users
```

## 🎯 Usage

### Daily workflow

```bash
# Morning — inside your project folder
cd ~/work/my-project
dstart

# code, commit, code, commit...

# End of day — same project folder
dend

# Daily note is filled in automatically!
```

### Multiple projects

Snapshots are **per project (folder name)**. Forgetting to `dend` project A before starting `dstart` in project B does not lose A's snapshot.

```bash
# Morning: project A
cd ~/work/project-a
dstart
# work...
dend       # appends "### 🔹 project-a" to the daily note

# Afternoon: project B
cd ~/work/project-b
dstart
# work...
dend       # appends "### 🔹 project-b" to the daily note
```

### Multiple wraps of the same project

```bash
cd ~/work/my-project
dstart

# Morning work...
dend       # 12:00 — adds the section (morning only)

# Afternoon work...
dend       # 18:00 — rewrites the same section (morning + afternoon)
```

> **How updates work**: the snapshot is frozen at `dstart`, so the second `dend` re-analyzes everything from morning to now and rewrites the whole section. Nothing from the morning is lost.

### Forgot to `dend` yesterday?

`dstart` detects it and asks.

```bash
cd ~/work/my-project
dstart

# ⚠️  Work from 2026-05-18 wasn't wrapped up!
#    - last dstart:    2026-05-18 09:12
#    - unsaved commits: 4
#
# 🤔 What would you like to do?
#    [1] Auto-run dend first (recommended)
#    [2] Handle manually
#    [3] Discard previous work ⚠️
#    [Enter] Cancel
```

### Already ran `dstart` today

A second `dstart` on the same day is safe.

```
# ⚠️  dstart already ran today!
#    [1] Keep existing snapshot (recommended)
#    [2] Run dend first (you'll be prompted)
#    [3] Force new snapshot ⚠️
#    [Enter] Cancel
```

### Token usage

`dend` logs to `~/.daily-note-automation/.token-usage.log`. Project key is the **current folder name** (`basename $(pwd)`).

```bash
dusage              # today + all-time summary
dusage today        # today details
dusage week         # last 7 days
dusage month        # last 30 days
dusage projects     # per-project totals
dusage all          # full history
```

## 🔧 How it works

### Git-snapshot-based change tracking

1. `dstart` saves a snapshot pointing at the **last commit before today midnight**
2. You work
3. `dend` collects everything from the snapshot to now
   - Committed: `git diff snapshot..HEAD`
   - Uncommitted: `git diff` (current working tree)
4. Claude Code summarizes the changes into meaningful units
5. The summary is written into the daily note under the project's section

### Daily-note section management

- First `dend` → adds `### 🔹 <project>` under `## 📝 Work Log`
- Re-running `dend` for the same project → replaces that section only
- `dend` for a different project → adds a separate section, leaving others alone

## 🐛 Troubleshooting

### `command not found: dstart`

```bash
source ~/.zshrc   # or ~/.bash_profile
```

Or fully close and reopen your terminal.

### "Current folder is not a git repository"

Run from your project root (the folder containing `.git`):
```bash
ls -la | grep .git
```

### "No snapshot found"

You need to run `dstart` before `dend`.

### Claude Code can't modify the note

May happen if the vault path contains special characters (e.g. spaces). Workaround: symlink.

```bash
ln -s "/path/with spaces/MyVault" ~/my-vault
```

Then set `VAULT="~/my-vault"` in `~/.daily-noterc`.

### Tokens show as 0

Install `jq`:
```bash
brew install jq      # macOS
sudo apt install jq  # Ubuntu
```

Or check your Claude Code version:
```bash
claude --version
claude -p "test" --output-format json | head -30
```

## 🗑 Uninstall

```bash
# remove scripts
rm -rf ~/.daily-note-automation
rm ~/.daily-noterc

# remove aliases: open ~/.zshrc (or ~/.bash_profile) and delete
# everything between
#   # >>> daily-note-automation >>>
#   # <<< daily-note-automation <<<
```

Your daily notes and code-change notes are **not deleted** (managed by you).

## 💰 Cost

Claude Code is billed per API usage. Rough averages:

- **1 `dend`**: ~$0.02 – $0.05 (15,000 – 50,000 tokens)
- **Per day** (3–5 runs): $0.10 – $0.20
- **Per month** (20 workdays): $2 – $5

Large diffs are truncated at 40,000 chars by default. Tune `MAX_DIFF_LEN` in `~/.daily-noterc`.
